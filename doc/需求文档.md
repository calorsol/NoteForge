# NoteForge 需求文档（PRD）

> 版本：v1.0　最后更新：2026-06-25
> 一个「资料归档 + 写作」二合一的个人在线写作辅助网站。

---

## 1. 产品概述

### 1.1 一句话描述
每天把收集到的资料、数据按"天"沉淀下来，再在一个漂亮、干净的写作中心里，把这些资料转化为成稿。

### 1.2 解决的问题
- 资料散落在各处（浏览器收藏、微信、备忘录），写作时找不到。
- 缺少一个把"收集"和"写作"连在一起的工作台。
- 想要登录后云端保存，换设备也能看到自己的资料和草稿。

### 1.3 目标用户
个人内容创作者 / 研究者 / 学生 —— 单人使用，每个账号的数据彼此隔离。

---

## 2. 核心概念（术语表）

| 术语 | 含义 |
|------|------|
| **资料（Material）** | 一份收集来的内容：一段文字、一组数据、一个链接摘要。资料库里每份资料是一个 Tab。 |
| **日期分组（Day）** | 资料按"采集日期"归组。每一天是资料库里的第一级 Tab。 |
| **文档（Document）** | 用户在写作中心里写的一篇文章/草稿。可有多篇。 |
| **用户（User）** | 一个账号，拥有自己全部的资料和文档。 |

## 3. 功能需求

### 3.1 账号系统（怎么简单怎么来）

**注册**
- 只需 **账号（用户名）+ 密码** 两个字段。
- 不需要邮箱、不需要手机号、不需要验证码、不需要邮件激活。
- 用户名唯一；若已存在则提示"用户名已被占用"。
- 密码在后端用 bcrypt 加密存储，**绝不明文存库**。
- 注册成功后自动登录，直接进入主界面。

**登录**
- 账号 + 密码登录。
- 登录后签发 JWT（或会话），前端保存，后续请求带上即视为已登录。
- 登录态有效期默认 7 天，过期重新登录。

**登出**
- 一键登出，清除本地登录态。

**约束**
- 用户名：2–32 个字符，字母/数字/下划线/中文。
- 密码：至少 6 位（弱口令也允许，主打简单，但给出最低限度）。

> 备注：这是单人/小范围使用的工具，不做找回密码、改密码以外的复杂账户功能（这些列为「未来可选」）。

### 3.2 资料库（两级 Tab）

**界面结构**
```
┌──────────────────────────────────────────────┐
│  日期 Tab： [ 06-25 ] [ 06-24 ] [ 06-23 ] ... │   ← 第一级：每天一个 Tab
├──────────────────────────────────────────────┤
│  资料 Tab： [ 资料A ] [ 资料B ] [ + 新增 ]    │   ← 第二级：当天每份资料一个 Tab
├──────────────────────────────────────────────┤
│                                                │
│   资料内容显示 / 编辑区                        │
│                                                │
└──────────────────────────────────────────────┘
```

**功能点**
1. **新增资料**：在某一天下新增一份资料，填写「标题 + 内容」。内容支持纯文本/Markdown（含数据、表格、链接）。
2. **日期 Tab**：
   - 默认按日期倒序排列（最新的一天在最前）。
   - 自动以"今天"作为默认选中的日期；新增资料若不指定日期则归到今天。
   - 允许查看历史任意一天。
3. **资料 Tab**：
   - 选中某一天后，展示当天所有资料的 Tab。
   - 点击某个资料 Tab，右侧/下方显示该资料内容。
4. **编辑资料**：可修改标题与内容并保存。
5. **删除资料**：删除单份资料（二次确认）。
6. **空状态**：某天没有资料时，显示引导"+ 新增第一份资料"。

**数据要求**
- 每份资料记录：所属用户、采集日期、标题、内容、创建时间、更新时间。

### 3.3 写作中心（漂亮、好用）

**界面结构**
```
┌───────────────┬──────────────────────────────┐
│  文档列表      │   文档标题：______________     │
│  • 文章一 ★    │  ┌────────────────────────┐   │
│  • 文章二      │  │                        │   │
│  • 草稿三      │  │   Markdown 编辑区       │   │
│  [ + 新建 ]    │  │   （可实时预览）        │   │
│               │  └────────────────────────┘   │
│               │   [自动保存 · 12:30 已保存]    │
└───────────────┴──────────────────────────────┘
```

**功能点**
1. **新建文档**：创建一篇新文档，带默认标题"无标题文档"。
2. **文档列表**：左侧列出当前用户的全部文档，按更新时间倒序。显示标题 + 更新时间。
3. **编辑器**：
   - Markdown 写作，**支持实时预览**（编辑/预览分栏或可切换）。
   - 干净、留白舒服、专注写作的排版（这是"漂亮"的重点）。
   - 字数统计。
4. **保存**：
   - **自动保存**：停止输入约 1–2 秒后自动保存（防丢）。
   - 也提供手动「保存」按钮 + 保存状态提示。
5. **删除文档**：二次确认后删除。
6. **（可选增强）从资料引用**：写作时能侧边查看资料库，把资料内容快速插入到正文。— 列为第二期。

**数据要求**
- 每篇文档记录：所属用户、标题、正文、创建时间、更新时间。

### 3.4 数据隔离与安全
- 所有资料、文档严格按 `user_id` 隔离，任何接口只能读写"当前登录用户"自己的数据。
- 后端对每个数据接口校验登录态与归属，禁止越权访问他人数据。

## 4. 非功能需求

| 维度 | 要求 |
|------|------|
| **易部署** | 本地一条命令即可启动；数据库用 SQLite 单文件，零额外安装。 |
| **响应速度** | 本地使用，常规操作 < 200ms。 |
| **数据安全** | 密码 bcrypt 加密；登录态有有效期；接口做归属校验。 |
| **美观** | 写作中心排版克制、留白舒适；整体风格统一、现代。 |
| **浏览器** | 支持现代浏览器（Chrome / Edge / Firefox 最新版）。 |
| **数据备份** | SQLite 数据库为单文件，复制该文件即完成备份。 |

## 6. 用户使用流程（典型场景）

1. **首次使用**：打开网站 → 注册（账号+密码）→ 自动登录 → 进入资料库。
2. **每天收集**：当天看到今天的 Tab → 点"+新增资料" → 粘贴一段资料/数据 → 保存。一天可加多份，每份一个 Tab。
3. **开始写作**：切到写作中心 → 新建文档 → 一边参考资料一边写 → 自动保存。
4. **次日回看**：登录 → 日期 Tab 切到昨天/前天 → 翻看历史资料。

## 7. 范围边界

**本期做（MVP）**
- 账号注册/登录/登出
- 资料库：日期 Tab + 资料 Tab + 增删改查
- 写作中心：文档增删改查 + Markdown 编辑 + 自动保存

**本期不做（明确排除）**
- 多人协作、分享、评论
- 邮箱/手机验证、第三方登录
- 富文本所见即所得编辑器（先用 Markdown）
- 移动端专门适配（先保证桌面可用）

**未来可选**
- 写作时从资料库一键引用/插入
- 资料全文搜索、标签
- 文档导出（PDF / Word / Markdown 文件）
- 修改密码、找回密码
- 深色模式

---

## 8. 已确认决策（2026-06-25 定稿）

1. 资料"日期"**允许手动指定**：新增资料时默认填今天，可改为任意日期以补录历史资料。
2. 写作中心的"引用/插入资料"功能 **放第二期**；第一期资料库与写作中心各自独立完成。
3. 深色模式 **放第二期**。
